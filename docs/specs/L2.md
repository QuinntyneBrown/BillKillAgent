# L2 Requirements — Bill Kill Agent UI Design

---

## L1-1: Onboarding Flow

### L2-1.1: Welcome / Sign-Up Screen
- Display app logo, tagline ("Stop overpaying. Start saving."), and sign-up options (email, Google, Apple)
- Include a "Log in" link for returning users
- **Acceptance**: Screen shows logo, tagline, three sign-up buttons, and login link. Responsive from 320px to 1440px+.

### L2-1.2: Connect Bank Accounts Screen
- Explain why bank connection is needed with trust signals (bank-grade encryption, read-only access, SOC 2)
- Show a "Connect with Plaid" primary CTA button and list of supported banks
- **Acceptance**: Screen shows security badges, explanation copy, Plaid CTA button, and bank logos. Responsive layout.

### L2-1.3: Scanning / Loading Screen
- Animated loading state showing transactions being scanned with progress indicator
- Display estimated time remaining ("Scanning your transactions...")
- **Acceptance**: Screen shows animated progress indicator with status text. Centered layout at all breakpoints.

### L2-1.4: Savings Discovery ("Aha Moment") Screen
- Large hero number showing total monthly savings found (e.g., "$347/month in savings found")
- Summary cards: number of subscriptions found, waste detected, potential negotiations
- Primary CTA: "Review & Save" button
- **Acceptance**: Hero savings number is prominent, summary cards visible, CTA button present. Responsive.

### L2-1.5: Action Approval Screen (Onboarding)
- List of recommended actions grouped by type (cancel, negotiate, switch)
- Each action shows subscription name, current cost, estimated savings, and approve/dismiss controls
- "Approve All" bulk action button and individual toggle switches
- **Acceptance**: Grouped action list with per-item controls and bulk approve. Responsive.

## L1-2: Dashboard Screen

### L2-2.1: Dashboard Header & Summary Cards
- Display greeting with user's name and current date
- Three summary cards: "Monthly Recurring Spend" (total), "Savings Found" (pending), "Savings Achieved" (realized)
- Each card shows amount and trend indicator (up/down arrow with percentage)
- **Acceptance**: Header with greeting, three summary cards with amounts and trends. Cards stack on mobile, row on desktop.

### L2-2.2: Savings Chart Widget
- Line or area chart showing savings achieved over the last 6 months
- Interactive tooltips on hover/tap
- **Acceptance**: Chart widget with 6-month data visualization. Responsive sizing.

### L2-2.3: Pending Actions Preview
- Show top 3-5 pending actions requiring approval with one-tap approve buttons
- "View All" link to Action Queue
- **Acceptance**: Action preview cards with approve buttons and "View All" link. Scrollable on mobile.

### L2-2.4: Recent Activity Feed
- Show last 5 activities (cancellations completed, negotiations finished, new charges detected)
- Each entry shows icon, description, timestamp, and outcome
- **Acceptance**: Activity feed with icons, descriptions, and timestamps. Responsive list layout.

## L1-3: Subscriptions List Screen

### L2-3.1: Subscription List Header & Filters
- Screen title "Your Subscriptions" with total count
- Filter bar: category filter (streaming, software, utilities, etc.), status filter (active, flagged, cancelled), sort options (cost, value score, date)
- Search input field
- **Acceptance**: Header with count, filter chips/dropdowns, search field. Collapses to icon filters on mobile.

### L2-3.2: Subscription Cards
- Each subscription shows: provider logo, name, monthly cost, billing date, category tag, value score (0-100 with color coding), usage indicator (bar or ring), and recommended action badge (cancel/negotiate/switch/keep)
- Tap/click to expand details
- **Acceptance**: Cards show all specified data points. Grid on desktop, stacked list on mobile.

### L2-3.3: Subscription Detail Expanded View
- Expanded view showing: full usage history (last 90 days), cost history, duplicate/overlap warnings, alternative suggestions, and action buttons (Cancel, Negotiate, Switch Plan, Dismiss)
- **Acceptance**: Detail view with usage chart, cost history, warnings, alternatives, and action buttons. Modal on mobile, inline expand on desktop.

## L1-4: Action Queue Screen

### L2-4.1: Action Queue Tabs
- Three tabs: "Pending Approval", "In Progress", "Completed"
- Badge count on each tab showing number of items
- **Acceptance**: Three tabs with badge counts. Swipeable on mobile.

### L2-4.2: Pending Action Cards
- Each pending action shows: subscription name/logo, action type (cancel/negotiate/switch), estimated savings, risk level, and approve/dismiss buttons
- Bulk "Approve All" button at top
- **Acceptance**: Action cards with all fields and approve/dismiss controls. Bulk approve button visible.

### L2-4.3: In-Progress Action Cards
- Each in-progress action shows: subscription name/logo, action type, current status (e.g., "Calling provider...", "Navigating cancellation flow..."), progress indicator, and estimated completion time
- **Acceptance**: Cards with real-time status indicators and progress bars.

### L2-4.4: Completed Action Cards
- Each completed action shows: subscription name/logo, action type, outcome (success/partial/failed), savings achieved, date completed, and "View Details" link
- **Acceptance**: Cards with outcome badges (green/yellow/red), savings amounts, and detail links.

## L1-5: Negotiation Log Screen

### L2-5.1: Negotiation Log List
- Chronological list of all negotiations with: provider name/logo, date, original rate, new rate, monthly savings, outcome badge (success/partial/failed)
- Summary stats at top: total negotiations, success rate percentage, total savings from negotiations
- **Acceptance**: List with all fields and summary stats header. Responsive layout.

### L2-5.2: Negotiation Detail View
- Full detail for a single negotiation: provider, date/time, duration, original rate, new rate, savings, negotiation transcript/summary, outcome, and any retention offers received
- **Acceptance**: Detail view with all negotiation data. Scrollable on mobile.

### L2-6.1: Savings Overview Header
- Large total savings number (lifetime) with monthly and annual breakdowns
- Savings destination display (e.g., "Redirected to: Fidelity Roth IRA")
- **Acceptance**: Prominent total savings, monthly/annual toggles, destination info. Responsive.

### L2-6.2: Savings Over Time Chart
- Area chart showing cumulative savings over time (monthly granularity)
- Toggle between "Savings Achieved" and "Projected Savings"
- **Acceptance**: Interactive area chart with toggle control. Full-width responsive.

### L2-6.3: Savings Breakdown
- Breakdown by category (streaming, utilities, insurance, etc.) with amounts and percentages

- **Acceptance**: Category breakdown with chart and amounts. Responsive layout.

### L2-6.4: Savings Transfer History
- List of all automated savings transfers with date, amount, destination account, and status
- **Acceptance**: Transfer history table/list with all fields. Scrollable on mobile.

## L1-7: Settings Screen

### L2-7.1: Connected Accounts Section
- List of connected bank accounts/credit cards with institution name, last 4 digits, connection status
- "Add Account" button and disconnect option per account
- **Acceptance**: Account list with status indicators, add/remove controls. Responsive.

### L2-7.2: Autonomy Level Selector
- Three-option selector: Supervised, Semi-Autonomous, Fully Autonomous
- Description of each level's behavior
- For semi-autonomous: configurable dollar threshold slider
- **Acceptance**: Three options with descriptions, threshold slider for semi-auto. Responsive.

### L2-7.3: Notification Preferences
- Toggle switches for: push notifications, email digests, SMS alerts
- Granular controls: price increases, trial expirations, negotiation outcomes, monthly reports
- **Acceptance**: Toggle switches organized by channel and event type. Responsive.

### L2-7.4: Savings Destination
- Connected investment/savings account display
- "Change Destination" button
- Auto-transfer toggle and frequency setting
- **Acceptance**: Destination display with change control and auto-transfer settings. Responsive.

### L2-7.5: Profile & Account
- User name, email, profile photo
- Change password, delete account, log out
- Premium tier upgrade CTA (if on free plan)
- **Acceptance**: Profile info, account actions, and premium upsell. Responsive.

## L1-8: Alerts & Notifications

### L2-8.1: Notification Center
- Bell icon in header with unread badge count
- Dropdown/panel showing recent notifications grouped by date
- Each notification: icon, title, description, timestamp, read/unread indicator
- **Acceptance**: Bell icon with badge, notification panel with grouped items. Responsive.

### L2-8.2: Alert Banners
- Contextual alert banners at top of relevant screens (e.g., "Netflix price increasing next month — want us to negotiate?")
- Dismissible with action button
- Color-coded by urgency (info blue, warning amber, critical red)
- **Acceptance**: Color-coded banners with dismiss and action buttons. Full-width responsive.

### L2-8.3: Monthly Report Card
- Monthly summary screen/modal: total saved, actions taken, new charges detected, top recommendation for next month
- Shareable format (for social proof / referral)
- **Acceptance**: Report card with summary stats and share button. Modal on desktop, full-screen on mobile.

## L1-9: Responsive Layout

### L2-9.1: Mobile Layout (320px - 767px)
- Single-column layout, bottom tab navigation, stacked cards, full-width components
- Hamburger menu for secondary navigation
- Touch-friendly tap targets (min 44px)
- **Acceptance**: All screens render correctly at 320px and 767px with bottom tabs and single-column layout.

### L2-9.2: Tablet Layout (768px - 1023px)
- Two-column layout where appropriate, bottom or side navigation
- Cards in 2-column grid
- **Acceptance**: All screens render correctly at 768px and 1023px with two-column grids.

### L2-9.3: Desktop Layout (1024px - 1439px)
- Sidebar navigation, multi-column content area, inline detail panels
- Cards in 3-column grid
- **Acceptance**: All screens render correctly at 1024px with sidebar nav and multi-column layout.

### L2-9.4: Large Desktop Layout (1440px+)
- Max-width container, generous whitespace, 3-4 column grids
- Side-by-side panels for list + detail views
- **Acceptance**: All screens render correctly at 1440px+ with max-width container and spacious layout.

### L2-10.1: Bottom Tab Bar (Mobile)
- Five tabs: Dashboard, Subscriptions, Actions, Savings, Settings

- Collapsible sidebar with: app logo, navigation links (Dashboard, Subscriptions, Action Queue, Negotiations, Savings, Settings), user avatar and name, notification bell
- Active state highlighting
- **Acceptance**: Sidebar with all nav links, user info, and active states at desktop breakpoints.

### L2-10.3: Top Header Bar
- App logo (mobile), page title, notification bell with badge, user avatar
- **Acceptance**: Header bar with contextual title and action icons. Responsive.
